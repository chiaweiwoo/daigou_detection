# CLAUDE.md — Claude Code Guide for daigou_detection

## Project objective

Detect probuyer-like / wholesale-like retail customers using HBOS anomaly detection on UCI Online Retail data. Provide LLM-generated explanations of structured DS evidence. Showcase on a Streamlit dashboard.

## Setup

```bash
uv python pin 3.11
uv sync
cp .env.example .env   # then fill in DEEPSEEK_API_KEY if you have one
```

## Run commands

```bash
# Full pipeline (download → clean → features → model → rules → explain → cases)
uv run python scripts/06_run_all.py

# Individual steps
uv run python scripts/01_download_data.py
uv run python scripts/02_prepare_data.py
uv run python scripts/03_train_model.py
uv run python scripts/04_generate_rules.py
uv run python scripts/05_generate_case_studies.py

# Dashboard
uv run streamlit run app/dashboard.py

# Tests
uv run pytest
```

## Known assumptions

- UCI Online Retail is a B2B-heavy dataset; many legitimate wholesale customers exist. We flag for review, not punishment.
- HBOS contamination is not set (unsupervised); risk bands are purely percentile-based.
- "Cancellation" features include returns (negative quantity). This is intentional.

## What to avoid

- Do not let the LLM decide risk scores or bands.
- Do not hardcode magic thresholds — derive from feature distributions or config constants.
- Do not crash the dashboard when the LLM is unavailable.
- Do not call customers "confirmed daigou" — say "probuyer-like" or "wholesale-like."
- Do not run notebooks (none in this project).
